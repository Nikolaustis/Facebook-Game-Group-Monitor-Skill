# V6.6.2 Incident Analysis

## Incident A: valid multi-game group removed

A Facebook group explicitly named both Anime Last Stand and Anime Vanguards. The previous collision resolver grouped all final rows only by `group_url`, compared match scores across games, and used `drop_all_tied` when the highest scores were equal. As a result, two valid game-specific rows were both removed.

This was a data-model error. A multi-game community is not a duplicate when the reporting dimension includes `game_name`.

V6.6.2 changes the uniqueness key to:

```text
group_url + game_name
```

One strongest row is retained for each matched game. Cross-game rows are not mutually exclusive.

## Incident B: finalized run did not shut down

The completed run contained a valid `phase2_autosave_state.json` with `finalized=true`, but the file was approximately 2 MB. The shutdown coordinator used Windows PowerShell 5.1 `ConvertFrom-Json` and silently returned `$null` on a parse exception. It then reported `checkpoint_finalized=false`, even though Node.js had already written and verified the checkpoint.

V6.6.2 moves authoritative shutdown-state parsing to Node.js, which already handles the package’s BOM-safe and large JSON files. The coordinator reads only a small derived verification report. Parse failures are retained in `read_errors`, verifier stderr, and coordinator status files.
