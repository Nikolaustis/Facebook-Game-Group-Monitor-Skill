# 监测流程标准（V2 严格版）

## 1) 登录阶段硬约束
- 必须先登录 Facebook，再执行后续动作。
- 登录前禁止搜索、采集、分析、报告输出。
- 登录提示固定句：
  - `请先完成 Facebook 登录，完成后回复：已登录。`

## 2) 字段采集硬约束
- 任何页面未明确出现的字段必须留空。
- 禁止猜测、推算、补全。
- `group_size` 只认成员数字段，不从无关数字匹配。

## 3) 相关性硬约束
- 不允许用“部分 token 命中”替代完整游戏名命中。
- 近似游戏名之间必须做归属裁决。
- 同一 `group_url` 只允许最终输出一条。

## 4) 区域锁定硬约束
- `region` 只允许由 `language_signal` 映射。
- Mixed / English / Unknown / Chinese 一律不落 TH/VN。
- 不允许从自由文本推断补位。

## 5) 输出顺序
1. 先输出 CSV 明细
2. 再输出中文摘要
3. 同时写出 `collision_report.json` 与 `audit_stats.json`

## 6) 禁止项
- 禁止编造数据
- 禁止把未达门槛记录写入输出
- 禁止在用户未确认“可以停止，继续”前进入第二轮
