# 监测流程标准（V3.6）

## 1. 登录阶段硬约束
- 必须先登录 Facebook，再执行后续动作。
- 登录前禁止搜索、采集、分析和报告输出。
- 登录提示固定为：`请先完成 Facebook 登录，完成后回复：已登录。`
- 用户回复“已登录”之前，不得进入第一轮或第二轮。

## 2. 第一轮硬约束
- 第一轮只使用用户输入的原始游戏名搜索。
- 不得扩展同义词、缩写、别名或地区词。
- 到达深翻停止条件后，必须询问用户：“可以停止，继续 / 继续深翻”。
- 用户未确认前，不得进入第二轮。

## 3. 第二轮硬约束
- 第二轮基于第一轮候选群组逐个进入详情页采集。
- 字段未在页面明确出现时必须留空。
- 禁止猜测、推算、补全字段。
- `group_size` 只认可成员数字段，不得从无关数字匹配。
- 语言识别必须以讨论区前五条玩家发言为主，先逐条识别再汇总；若前五条出现两个以上可信语言，标记为 `Mixed`。无正文帖、图片/视频帖、极短正文帖不得从 Facebook 中文界面文案中推断为 `Chinese`。
- 群组名称为语言辅助，用户手写 about 文本为最低优先级。
- 地区识别必须优先依据群组名称中的明确地区语义，语言只能作为辅助。

## 4. 相关性硬约束
- 不允许用“部分 token 命中”替代完整游戏名命中。
- 近似游戏名之间必须做归属裁决。
- 同系列兄弟游戏在 `group_name` 中明确命中时，必须排斥。
- 同一 `group_url` 只允许最终输出一条。

## 5. 输出硬约束
- V3.6 不读写、不保存 CSV。
- 最终只保存 Excel 工作簿 `fb_monitoring_filtered.xlsx`。
- Excel 工作簿必须包含 `detail` 工作表和 `manual_review` 工作表。
- `detail` 只保留严格通过样本。
- `manual_review` 只保留人工复核候选，不得混入 `detail`。
- 同时保存 `fb_monitoring_filtered_summary.json`、`collision_report.json`、`audit_stats.json`。

## 6. Excel 格式硬约束
- `snapshot_date` 必须以文本日期保存，例如 `2026-05-07`，不得变成 Excel 序列号。
- `group_id` 必须按文本格式保存，避免长数字科学计数法或乱码。
- `活跃指数=当日新帖/社群规模` 和 `规模增速=上周新增/(社群规模-上周新增）` 必须为公式列。
- 活跃指数和规模增速单元格格式必须为百分比，保留 2 位小数。
- 非 ASCII 群名，例如泰语、越南语、老挝语、缅甸语、柬埔寨语等，必须在 XLSX 中保持可读。

## 7. 禁止项
- 禁止编造数据。
- 禁止把未达门槛记录写入 `detail`。
- 禁止把人工复核候选混入最终严格通过样本。
- 禁止使用 Facebook 账号界面语言作为群组语言证据。
- 禁止在用户未确认“可以停止，继续 / 继续深翻”前进入第二轮。
- 禁止通过 CSV 中转生成最终文件。
