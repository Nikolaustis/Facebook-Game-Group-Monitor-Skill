# V6.3.0 覆盖说明

本补丁基于 V6.2.1，解决“用户必须手动修改固定关机日期或配置文件”的问题。

## 新规则

- 默认完成后不关机。
- 用户只需在 Codex 文本框中说明是否关机，以及需要时的截止时间。
- Codex 将用户原话转换为 `ShutdownMode` 和动态截止时间；启动脚本自动写入当前运行目录的 `shutdown_policy.json`。
- 用户无需编辑 `task_config.json`、`shutdown_policy.json` 或 PowerShell 文件。
- 截止时间不会写死在 Skill 中，每次任务根据当前提示词重新解析。
- 系统重启续跑、任务计划程序和 direct fallback 使用同一策略。
- 运行过程中重新说明关机要求时，Codex 可用同一 `RunDir` 再次提交模式；最终完成前会重新读取策略。
- 正常结束后主计划任务和关机兜底任务继续自动删除。

## Codex 使用方式

用户未提关机或明确要求不关机：

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json" -ShutdownMode none -ShutdownInstruction "用户未要求关机"
```

用户要求完成后立即关机：

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json" -ShutdownMode after_complete -ShutdownDelaySeconds 0 -ShutdownInstruction "完成后立即关机"
```

用户要求截止时间前完成才关机：

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json" -ShutdownMode before_deadline -ShutdownDeadline "<Codex 根据用户提示解析出的带时区绝对时间>" -ShutdownDelaySeconds 0 -ShutdownInstruction "<用户原话摘要>"
```

覆盖到 Skill 根目录并替换同名文件即可，无需重新安装 npm 依赖。
