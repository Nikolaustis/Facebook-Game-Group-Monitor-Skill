# Facebook Group Monitor Skill V6.0.0 覆盖说明

## 本版本解决

- 第二轮默认交给 Windows【任务计划程序】运行，Windows 更新或系统重启中断后，在用户重新登录时自动续跑。
- 每处理一个候选都完整保存 detail 暂存、manual_review、stats 和游标，恢复位置不会超过完整 checkpoint。
- 运行期间保持系统唤醒，并持续取消可撤销的待执行关机/重启。
- 只有全部最终文件和 finalized/completion 校验通过后才允许完成后关机；任何失败都保持开机。
- 每次任务计划程序正常执行结束后立即自删除，不保留冗余任务；因重启中断时暂时保留以便续跑，续跑结束后删除。

## 覆盖方法

1. 停止当前第二轮 Node/PowerShell 进程。
2. 若任务计划程序中已经存在旧测试任务，可先删除名称以 `FBGroupMonitor_Phase2_` 开头且确认不再运行的任务。
3. 将覆盖包解压到现有 Skill 根目录，替换同名文件。
4. 不需要重新执行 `npm install`。
5. 使用原 `RunDir`、`Index` 和配置重新运行 `phase2:bg`。V6.0 会从完整 autosave 安全续跑。

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json" -ShutdownAfterComplete -ShutdownDelaySeconds 60
```

查看状态：

```powershell
npm run status:bg -- -RunDir ".\runs\demo"
```

## 注意

- V6.0 的 power guard 是用户态保护，无法绝对阻止断电、固件、管理员强制重启、内核故障或不可撤销的 Windows Update 重启。
- 它会阻止睡眠并尽可能取消有倒计时、可撤销的关机/重启；同时逐候选完整 checkpoint 与登录后自动续跑确保即使重启仍可恢复。
- 不要在同一 `RunDir` 上同时使用 `phase2:bg` 和 `phase2:direct-bg`。

# Facebook Group Monitor Skill V5.8.0 覆盖说明

## 本补丁解决

- GeoNames `alternateNames` 含对象时不再因 `.replace is not a function` 终止第二轮。
- 同一运行目录存在未完成 autosave 时，重新执行普通 `phase2:bg` 命令会自动从断点继续。
- V5.8.0 曾合并逐候选 progress 游标；V6.0 已改为仅按完整 autosave 游标恢复，避免跳过未保存的人工复核数据。

## 覆盖方法

1. 停止正在运行的第二轮进程。
2. 将覆盖包解压到现有 Skill 根目录，替换同名文件。
3. 不需要重新执行 `npm install`。
4. 使用原来的 `RunDir` 和 `Index` 再次运行 `phase2:bg`，默认会自动续跑。

需要从头重跑时使用：

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json" -FreshStart
```

## 自检

```powershell
node --check .\scripts\phase2_collect_details.js
```

运行日志在恢复时会打印 `phase2_resume: true` 和恢复游标。

# Facebook Group Monitor Skill V5.7.0 覆盖说明

## 覆盖方法

1. 停止正在运行的第二轮任务。
2. 将本压缩包解压到现有 Skill 根目录，选择“替换同名文件”。
3. 不需要重新执行 `npm install`。
4. 重新运行第二轮；已经运行完成的历史结果不会自动重算。

## 本版重点

第二轮会在访问 Facebook `/about` 和讨论页前，先读取第一轮候选中的 `group_name`：

- 明确命中目标标题/别名/受控变体：继续；
- IP root 或兄弟标题弱命中：默认继续进入人工复核链路；
- 完整群名完全无关：直接跳过并节省页面访问；
- seed URL、缺失或截断群名：继续打开 About，避免漏采。

默认配置：

```json
"phase2_name_prefilter": {
  "enabled": true,
  "allow_manual_review_candidates": true,
  "treat_incomplete_as_inconclusive": true
}
```

若临时需要恢复旧行为，可在任务配置中设置：

```json
"phase2_name_prefilter": { "enabled": false }
```
