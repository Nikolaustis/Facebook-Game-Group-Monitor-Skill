# Facebook Group Monitor Skill V4.1.0 覆盖说明

## 本补丁新增内容

当原有地区判断链路仍无法得到 `region` 时，第二阶段会从已经打开的群组 About 页面中读取明确标注的“所在地 / Location”字段，并按以下**末级兜底**顺序判断：

1. 群组名称中的国家/地区/属地/大区语义。
2. 同一业务大区内多国命中的归并规则；跨区冲突时暂不归类。
3. 允许的高确定性语言映射。
4. About 的所在地：先识别国家/地区或大区；未命中时，再识别高确定性城市并映射到既有地区输出规则。

About 所在地不会覆盖前 1–3 步已经得到的 `region`。但当群名因跨大区关键词冲突而留空时，明确的 About 所在地可以作为最终可信位置证据解决该冲突。

## 覆盖方法

1. 关闭正在运行的第一轮或第二轮监测任务。
2. 将本压缩包**解压到现有 Skill 根目录**，选择“替换同名文件”。
3. 本补丁没有新增 npm 依赖，不需要重新执行 `npm install`。
4. 用旧任务配置运行也可以：核心城市表已经内置。需要扩展城市识别时，再把 `assets/task_config.template.json` 中的 `about_location_city_keywords` 按需合并进自己的任务配置。

## 新增审计字段

- `__region_location`：从 About 页“所在地 / Location”字段解析出的原始位置文本。
- `__region_source` 新增：
  - `about_location_country_keyword`
  - `about_location_city_keyword`
  - `about_location_region_keyword`
- `__region_keyword_hits` 在 About 兜底介入时，会区分 `group_name:` 与 `about_location:` 两类证据。
- `manual_review` 工作表新增 `about_location` 列。
