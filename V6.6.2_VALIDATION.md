## Multi-game regression

Input rows used the same URL with:

```text
Anime Last Stand, score 314
Anime Vanguards, score 314
Anime Last Stand, score 300 duplicate
```

Expected and observed:

- Anime Last Stand score 314 retained.
- Anime Vanguards score 314 retained.
- Lower-scoring Anime Last Stand duplicate removed.
- Resolution recorded as `keep_each_matched_game`.
- `multi_game_groups_preserved=1`.
- `multi_game_rows_preserved=2`.
- `same_game_duplicate_rows_dropped=1` for that URL.

The same regression passed for the live collector and recovery finalizer implementations.

## Large checkpoint shutdown regression

The Node verifier was run against the uploaded completed run containing a `phase2_autosave_state.json` of 2,095,653 bytes.

Observed:

```text
checkpoint_readable=true
checkpoint_finalized=true
progress_finalized=true
all_valid=true
read_errors=[]
```

This confirms that the shutdown decision no longer depends on Windows PowerShell 5.1 parsing the full checkpoint.

## Environment limitation

The container cannot execute Windows Task Scheduler, Windows PowerShell 5.1, or an actual system shutdown. PowerShell integration was statically inspected; the Node verifier and collision behavior were executed directly. Final Task Scheduler and `shutdown.exe` behavior must be confirmed on the user’s Windows machine.
