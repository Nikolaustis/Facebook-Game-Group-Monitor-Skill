# V6.6.0 Patch Notes

- Added one shared BOM- and encoding-safe JSON reader for active JavaScript workflows.
- Phase 1 and phase 2 now accept UTF-8 BOM and UTF-16 JSON generated by Windows tooling.
- Added `validate_phase2_inputs.js` to parse the index, config, shutdown policy, and every candidate file before launch.
- Added `phase2_supervisor.js`; a phase-2 task is reported as started only after a fresh, readable progress checkpoint appears.
- Startup failures now record child PID, exit code, signal, spawn error, and stderr tail.
- Added `queue_phase2_after_current.ps1` for reliable queued execution after another Facebook task.
- The handoff writes explicit status, validates inputs, verifies actual startup, and can retry launch failures.
- Core PowerShell JSON writers now emit UTF-8 without BOM.
- Preserved the uploaded base-package XLSX field order without reordering output columns.
- Retained API-first semantic adjudication, Codex CLI fallback, GeoNames controls, full per-candidate checkpoints, scheduled recovery, task self-deletion, and prompt-driven shutdown.
