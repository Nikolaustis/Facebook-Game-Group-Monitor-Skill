# V6.0.0 验证说明

## 目标

V6.0.0 解决以下问题：

1. Windows Update 或系统重启中断第二轮后，能够在用户重新登录 Windows 时自动续跑。
2. 完整恢复状态必须在每个候选处理后原子写入，恢复游标不得超过完整数据 checkpoint。
3. 第二轮运行期间保持系统唤醒，并持续尝试取消待执行的用户态关机/重启。
4. 只有最终 XLSX、summary、collision、audit、debug rows、finalized checkpoint、finalized progress 和 completion token 全部通过校验后，才允许执行完成后关机。
5. 每个任务计划程序任务在正常执行结束后立即自删除；若执行被系统重启打断，任务保留并在下次用户登录时续跑，续跑正常结束后再删除。

## 静态验证

```powershell
node --check .\scripts\phase2_collect_details.js
node --check .\scripts\conditional_shutdown_watcher.js
```

应无语法错误。

## 任务计划程序验证

启动第二轮：

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json"
```

检查：

```powershell
Get-ScheduledTask | Where-Object TaskName -like "FBGroupMonitor_Phase2_*"
Get-Content -Raw .\runs\demo\background_task.json
Get-Content -Raw .\runs\demo\scheduled_phase2_runner_status.json
```

预期：

- `launch_mode = windows_task_scheduler`；
- 任务包含立即启动与 `AtLogOn` 恢复能力；
- 同一 `RunDir` 不创建并行重复实例；
- 正常结束后任务消失，`scheduled_task_deleted = true`；
- `scheduled_phase2_manifest.json` 在任务删除成功后同步删除。

## 完整 checkpoint 验证

连续观察：

```powershell
Get-Content -Raw .\runs\demo\phase2_autosave_state.json
Get-Content -Raw .\runs\demo\phase2_progress.json
```

预期：

- 每处理一个候选，`phase2_autosave_state.json` 的 `updated_at`、游标、`staged_rows`、`manual_review_rows` 和 stats 同步刷新；候选中途异常时游标不得越过该候选；
- `autosave_mode = full_json_every_candidate_xlsx_on_accepted_row`；
- 恢复时 `phase2_resume_source = full_autosave_checkpoint`；
- `phase2_progress.json` 只用于观察，不得推动恢复游标超过完整 checkpoint。

## 电源保护验证

运行期间检查：

```powershell
Get-Content -Raw .\runs\demo\runtime_power_guard_status.json
```

预期：

- `status = active`；
- `prevents_sleep = true`；
- `cancels_pending_shutdown = true`。

说明：该保护能阻止睡眠并反复调用 `shutdown.exe /a` 取消可撤销的待执行关机/重启，但不能覆盖固件、断电、管理员强制操作、内核故障或不可撤销的 Windows 更新重启。

## 严格关机门槛验证

启用完成后关机：

```powershell
npm run phase2:bg -- -Index ".\runs\demo\phase1_index.json" -RunDir ".\runs\demo" -Config ".\runs\demo\task_config.json" -ShutdownAfterComplete -ShutdownDelaySeconds 60
```

预期只有以下条件全部为真时才发送关机命令：

- 最终 XLSX 大于 1 KB；
- summary、collision、audit、debug rows 文件存在；
- `phase2_autosave_state.json.finalized = true`；
- `phase2_progress.json.finalized = true`；
- `codex_task_complete.json.phase2_finalization_verified = true`；
- Chrome 已确认关闭；
- completion token 与 watcher token 一致。

任何一项失败时，`conditional_shutdown_watcher_status.json` 应记录 `shutdown_not_requested_validation_failed`，电脑保持开机。
